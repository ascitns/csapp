## 5.04 消除循环的低效率

### 什么是要消除的

循环体内每次迭代都反复计算同一个值，但结果在循环期间根本没变过。这个值就是**循环不变量**。把它提到循环前面算一次，就是**代码移动（code motion）**

```c
// 问题：循环条件里的 vec_length(v) 每轮都调用一次
for (i = 0; i < vec_length(v); i++) {
    // 处理 v->data[i]
}
```

`vec_length` 的典型实现是读一个结构体字段：

```c
long vec_length(vec_ptr v) {
    return v->len;   // 就一行
}
```

但这一行藏在一个函数调用里。函数调用在汇编层面 `call` ->开辟栈帧 -> `ret`，循环 n 次就执行 n 遍

修复后：

```c
long length = vec_length(v);   // 只调一次，结果放到局部变量（寄存器）
for (i = 0; i < length; i++) { // 循环条件变成：比较两个寄存器
    // 处理 v->data[i]
}
```

- `strlen` 内部是一个循环，从头扫描到 `\0`。把这个循环放在外循环的每次迭代里，时间复杂度为 O(n²) 

- GCC 在 `-O2` 下**能**识别 `strlen` 并自动做代码移动，因为它知道标准库函数的语义

### 不能将代码移动的情况

代码移动的前提是：被移动的表达式**真的是**循环不变量

| 不能提的场景 | 原因 |
| :--- | :--- |
| 循环内修改了数组或结构体 | 长度可能在迭代中变化 |
| 被调函数依赖循环内修改的全局状态 | 其他迭代可能改变了该状态 |
| 被调函数有副作用（日志、计数） | 移动后会改变程序的可观测行为，这是 bug |

*code review 时检查循环条件里有没有函数调用*
