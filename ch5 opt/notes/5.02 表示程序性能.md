## 5.02 表示程序性能

### 为什么用 CPE

- 处理器活动的顺序由**时钟**控制
- **时钟**提供某个频率的规律信号，通常用 **GHz**（Gigahertz，千兆赫兹，十亿周期每秒）表示
- 一个 **4 GHz** 的处理器，**时钟周期**为 **0.25 纳秒**

用**时钟周期**度量性能比用纳秒或皮秒更有意义——度量值反映执行的指令数量，不反映时钟频率

引入 **CPE**（Cycles Per Element，每元素周期数）：处理数组中一个元素平均花费的 **CPU 周期数**

`CPE = 总周期数 / 元素数量`

- **CPE** 独立于 **CPU 频率**——同一微架构上同一算法的 **CPE** 是常数
- **墙钟时间**（Wall Clock Time）受机器频率、后台进程干扰，换机器就无法比较
- **CPE** 控制住频率和数据规模两个变量，只比较代码与微架构的关系

### 用**前置和**（Prefix Sum）展示 **CPE** 的测量

对向量 **a** = (a₀, a₁, ..., aₙ₋₁)，前置和 **p** = (p₀, p₁, ..., pₙ₋₁)：

```
p₀ = a₀
pᵢ = pᵢ₋₁ + aᵢ    (1 ≤ i < n)
```

**psum1** 每次迭代计算一个元素：

```c
/* Compute prefix sum of vector a */
void psum1(float a[], float p[], long n)
{
    long i;
    p[0] = a[0];
    for (i = 1; i < n; i++)
        p[i] = p[i-1] + a[i];
}
```

**psum2** 使用**循环展开**（Loop Unrolling），每次迭代计算两个元素：

```c
void psum2(float a[], float p[], long n)
{
    long i;
    p[0] = a[0];
    for (i = 1; i < n-1; i+=2) {
        float mid_val = p[i-1] + a[i];
        p[i]   = mid_val;
        p[i+1] = mid_val + a[i+1];
    }
    if (i < n)
        p[i] = p[i-1] + a[i];
}
```

- **循环展开**用更少的循环次数完成等量计算，每次迭代处理多个元素

对两个函数建立坐标轴进行统计

每个数据点对应一个 **n** 值，测量函数消耗的总周期数 **T**

对数据做**最小二乘拟合**（Least Squares Fit），发现：

```
psum1: T ≈ 368 + 9.0 · n     → CPE = 9.0
psum2: T ≈ 368 + 6.0 · n     → CPE = 6.0
```

- **368** 周期是固定开销——代码计时、初始化、准备循环、完成过程的常数部分
- **9.0n** / **6.0n** 是与 **n** 成正比的线性部分，斜率就是 **CPE**
- **n** 较大（如 >200）时，运行时间由线性因子主导
- **psum2** 的 **CPE = 6.0** 优于 **psum1** 的 **CPE = 9.0**
