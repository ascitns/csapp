## 3.05 算术和逻辑操作

### 3.05.1 加载有效地址

- `leaq` 指令的全称是 Load Effective Address，字面意思是"加载有效地址"
- 根据内存寻址的表达式 `Imm(rb, ri, s)` 计算出一个地址，然后把**这个地址本身**（不是该地址处的内存内容）存入目标寄存器

*因为这个特性所以这本书上的leaq指令基本用来算术运算了*

```asm
leaq    Src, Dst    # Dst ← &Src（只计算地址，不访问内存）
```

- Src 必须写成内存寻址的形式，Dst 必须是寄存器

### 3.05.2 一元和二元操作

**一元操作：**

| 指令 | 效果 | 说明 |
| :--- | :--- | :--- |
| `incq D` | D ← D + 1 | ++                       |
| `decq D` | D ← D - 1 | - - |
| `negq D` | D ← -D | 取负（**补码取反加一**） |
| `notq D` | D ← ~D | 按位取反 |

**二元操作：**

| 指令 | 效果 | 说明 |
| :--- | :--- | :--- |
| `addq S, D` | D ← D + S | 加法 |
| `subq S, D` | D ← D - S | 减法 |
| `imulq S, D` | D ← D × S | 整数乘法（会**截断**） |
| `andq S, D` | D ← D & S | 按位与 |
| `orq S, D` | D ← D \| S | 按位或 |
| `xorq S, D` | D ← D ^ S | 按位异或 |

- **D和S不能同时为内存** 如果需要操作两个内存位置的值，至少要把其中一个先加载到寄存器 

- `xorq %rax, %rax` 会把`%rax`**置零** *这个在很多课后题都考过*

### 3.05.3 移位操作

| 指令 | 效果 | 说明 |
| :--- | :--- | :--- |
| `salq k, D` | D ← D << k | **算术**左移 |
| `shlq k, D` | D ← D << k | **逻辑**左移 |
| `sarq k, D` | D ← D >>_A k | **算术**右移 |
| `shrq k, D` | D ← D >>_L k | **逻辑**右移 |

- `salq` 和 `shlq` 的效果相同

- **移位量** k 的来源有两种：要么是一个**立即数**，要么是 `%cl`（这是 x86 的硬性规定）。
- 实际移位量会按**操作数宽度**自动取模（例如对 64 位操作数只取移位量的低 6 位，即模 64）

### 3.05.5 特殊的算术操作

```asm
imulq   S       # 有符号全乘法：S × %rax → %rdx:%rax（128 位结果）
mulq    S       # 无符号全乘法：S × %rax → %rdx:%rax
```

- 这两条指令只有一个操作数 S（可以是寄存器或内存），另一个操作数隐式地固定在 `%rax`

- 除法是乘法的逆操作 一个 128 位的被除数 ÷ 64 位的除数，结果是一个 64 位的商和一个  64 位的余数 

```asm
idivq   S       # 有符号除法：%rdx:%rax ÷ S → 商=%rax，余数=%rdx
divq    S       # 无符号除法：%rdx:%rax ÷ S → 商=%rax，余数=%rdx
```

- **在做除法之前，必须先把 `%rax` 中的 64 位被除数扩展到 128 位**（填满 `%rdx`）
- **有符号**除法，用 `cqto` 指令将 `%rax` 符号扩展到 `%rdx:%rax`；
- **无符号**除法，直接 `xorq %rdx, %rdx` 把 `%rdx` 清零即可 

```asm
# 计算 x / y（x 在 %rax 中，y 在 %rsi 中）——有符号
cqto                    # 将 %rax 符号扩展到 %rdx:%rax
idivq   %rsi            # %rdx:%rax / %rsi → 商在 %rax，余数在 %rdx

# 计算 x / y——无符号
xorq    %rdx, %rdx      # 清零 %rdx（无符号扩展）
divq    %rsi            # %rdx:%rax / %rsi → 商在 %rax，余数在 %rdx
```
