## 3.01 历史观点

### 3.01.1 Intel 处理器架构演进

*最没意思的一节*

| 处理器 | 年份 | 晶体管数 | 关键特性 |
| :--- | :--- | :--- | :--- |
| 8086 | 1978 | 29K | 16 位架构，x86 的起点 |
| i386 | 1985 | 275K | **32 位**架构，平坦寻址模型 |
| i486 | 1989 | 1.2M | 片上浮点单元（FPU），流水线 |
| Pentium | 1993 | 3.1M | 超标量，分支预测 |
| PentiumPro | 1995 | 5.5M | **乱序执行**，微架构革命 |
| Pentium/II/III | 1997–1999 | 7M–28M | SSE 指令集（单指令多数据） |
| Pentium 4 | 2000–2005 | 42M–125M | NetBurst 微架构，超线程 |
| Core 2 | 2006 | 291M | 多核，回归高效能设计 |
| Core i7 (Nehalem) | 2008 | 731M+ | 集成内存控制器，超线程回归 |

### 3.01.2 指令集架构（ISA）的概念

- 每一代处理器虽然在内部实现上完全不同（流水线深度、缓存大小、执行单元数量都不一样），但都支持**同一套指令集**。这个指令集就是处理器和软件之间的**抽象接口**，称为 **ISA（Instruction Set Architecture，指令集架构）**。

- ISA 定义了程序员（或编译器）能够看到的处理器行为：有哪些寄存器、每条指令的编码是什么、内存如何寻址。至于处理器内部是怎么执行这些指令的——是顺序执行还是乱序执行、有几个加法器——这些属于**微架构**的范畴，对 ISA 是不可见的。

### 3.01.3 x86-64 的诞生

两千年初，32 位架构的两个限制变得无法忽视：

- 服务器和大型数据库需要更大的内存
- **8 个通用寄存器**让编译器在优化时捉襟见肘

2003 年，**AMD** 率先打破了局面。AMD 在 x86 的基础上扩展出了 64 位架构，命名为 **AMD64**：

1. **寄存器宽度**从 32 位扩大到 **64 位**，同时保留了 32 位和 16 位操作模式，兼容旧程序
2. **通用寄存器数量**从 8 个增加到 **16 个**（新寄存器被命名为 `%r8` 到 `%r15`）。这意味着函数调用时可以更多地用寄存器传递参数，而不必反复访问内存。
3. **虚拟地址空间**从 4GB 扩展到理论上的 256TB（实际实现中通常只使用低 48 位）。

- Intel 随后也采纳了这套方案，命名为 **Intel 64**。现在我们说的 **x86-64**（或简写为 x64）就是指这套共同标准。

- 本书后续所有讨论都基于 x86-64。寄存器名称以 `%r` 开头（如 `%rax`、`%r8`），指针是 8 字节宽，这些都是 x86-64 的基本特征。
