## 3.02 程序编码

### 3.02.1 机器级代码

切换到机器级编程的视角时，有几个重要的思维转变需要建立：

-  `%rip` 始终指向下一条要执行指令的地址。这在后面过程一节的**调用**中很常见——`call` 之后的**返回地址**正是下一条指令。

- 类型、数组、结构体都是高级语言的抽象，机器只知道地址和字节

- **寄存器**，**内存**，**立即数**

- 你在 `objdump -d` 输出里看到的 `<multstore>` 这样的标签，是反汇编工具为增强可读性而添加的（我第一次见到时困惑了很久）。

### 3.02.2 代码示例

- 下面的流程图展示了 gcc 从 C 源码生成可执行文件的完整路径：

```text
C 源码 (.c)
  │  gcc -Og -S          ← 编译器（C → 汇编）
  ▼
汇编代码 (.s)
  │  gcc -Og -c          ← 汇编器（汇编 → 机器码）
  ▼
目标文件 (.o)
  │  gcc -Og -o          ← 链接器（多个 .o + 库 → 可执行文件）
  ▼
可执行文件
```

- **objdump** 是 Linux 下最常用的反汇编工具，它能把目标文件里的机器码字节翻译回人类可读的汇编指令：

```bash
gcc -Og -c mstore.c          # 编译+汇编，生成目标文件 mstore.o
objdump -d mstore.o          # 反汇编
```

输出的典型形式如：

```text
0000000000000000 <multstore>:
   0:   53              push   %rbx
   1:   48 89 d3          mov    %rdx, %rbx
   ...
```

*后续章节中会反复出现这种输出格式*

### 3.02.3 关于格式的注解

| 特性 | AT&T 格式 | Intel 格式 |
| :--- | :--- | :--- |
| 操作数顺序 | `op 源, 目的` | `op 目的, 源` |
| 寄存器前缀 | `%rax` | `rax` |
| 立即数前缀 | `$42` | `42` |
| 操作数大小 | 指令后缀表示：`movl` | 由操作数推断，或加 `dword ptr` |
| 内存寻址 | `8(%rax,%rcx,4)` | `[rax + rcx*4 + 8]` |

- 本书使用 **AT&T** 格式，这也是 **GCC** 和 **objdump** 的默认输出格式。

*整体上 Intel 格式的逻辑更直观，风格也更现代*
