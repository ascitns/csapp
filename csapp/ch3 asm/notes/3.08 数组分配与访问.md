## 3.08 数组分配与访问

### 3.08.1 基本原则

- **数组的声明**：**`T A[N]`**，效果有两个：

1. 在**内存**中分配一个 **L × N** 字节的连续区域（L 是数据类型 **T** 的字节大小）
2. 引入标识符 A，A 在表达式中可以作为指向数组**第一个**元素的**指针**，即 `&A[0]`

*数组的元素通过**地址 + 偏移量**寻找，数组元素 **i** 将存放在地址为 **A + L × i** 的地方。*

### 3.08.2 指针运算

- 指针运算本质上就是 `leaq` 及寻址语法的应用。

### 3.08.3 嵌套数组

对于数组声明 **`T D[R][C]`**

- **`&D[i][j]` = D + L * (C * i + j)**

*这一节的习题主要利用上述公式反推数组参数。*

### 3.08.4 定长数组

- 基于数组的内存连续性，一种常见的二维数组优化方式是：创建行指针和列指针，通过直接增减指针值来遍历元素。由上述公式可知，如果直接用 for 循环遍历，每次迭代都需要重新计算元素地址并求值，开销较大。

*本节还涉及 `#define` 宏定义和常见的数组遍历模式。*

### 3.08.5 变长数组

- 由嵌套数组公式可知，列数 C 在**声明**时必须能够确定，而行数 R 可以不明确。
- `[]` 内的值可以是表达式。

  ```c
  int var_ele(long n, int A[n][n], long i, long j)
  {
      return A[i][j];
  }
  ```

- 在动态声明中，数组元素寻址通常需要通过**乘法指令**对索引 **i** 进行缩放，这会**拖慢**性能。
- **GCC** 可以识别出多维数组元素的**步长**并进行优化，从而显著提高性能。
