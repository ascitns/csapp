## 5.05 减少过程调用

### 问题和优化方法

- 即使是最简单的函数，对编译器来说也是一个**黑盒**，不知道其调用前后的改动和实际效果

*具体到举例*

```c
for (i = 0; i < length; i++) {
    data_t val;
    get_vec_element(v, i, &val);   // 每轮一次函数调用
    *dest = *dest OP val;
}
```

`get_vec_element`的功能有两个：检查边界和取值，但注意到**本身数组下标已经被限制在length**内，做边界检查是没有意义的

且`get_vec_element` 在另一个 `.c` 文件里，当前编译单元看不到它的实现，必须遵守调用函数的规则

按 ABI 规范把参数塞到 `%rdi`、`%rsi`、`%rdx`，执行 `call` 指令，采用保守策略猜测，不**重排**后续循环操作和函数调用之间的顺序（上一节）

- 对此的优化方法：删了

```c
data_t *data = get_vec_start(v);  
for (i = 0; i < length; i++) {
    *dest = *dest OP data[i];   
}
```

消除函数调用后，循环体对编译器来说就是一个连续的基本运算，可以进行优化

*减少过程调用本身并不一定会带来巨大 CPE 下降，但是可以增加后续优化的可能，使编译器可以更加激进*
