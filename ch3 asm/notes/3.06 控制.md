## 3.06 控制

### 3.06.1 条件码

| 条件码 | 全称 | 何时被置 1 |
| :--- | :--- | :--- |
| **CF** | Carry Flag（进位标志） | **无符号**运算产生了进位或借位 |
| **ZF** | Zero Flag（零标志） | 运算结果为 **0** |
| **SF** | Sign Flag（符号标志） | 结果的**最高位**为 1（在补码中最高位就是符号位） |
| **OF** | Overflow Flag（溢出标志） | **有符号**运算发生了溢出 |

- **大部分算术和逻辑指令都会更新条件码**：`addq`、`subq`、`xorq`、`incq` 等指令执行后，条件码会根据运算结果被重新设置。
- **`leaq` 不修改任何条件码**（它本质上不是算术指令，尽管常被用于算术运算），`mov` 指令也**不修改任何条件码**。

### 3.06.2 访问条件码

- `cmpq` 和 `testq` **计算但不存储**：

```asm
cmpq    S2, S1   
testq   S2, S1    
```

- `cmpq` 等价于 `subq`，但丢弃结果。

- `testq` 等价于 `andq`，但丢弃结果。它最常见的用途是**判断一个寄存器是否为零**——`testq %rax, %rax` 计算 `%rax & %rax`，结果永远等于 `%rax` 本身，所以如果 `%rax` 是 0，ZF 就是 1；如果 `%rax` 的最高位（符号位）是 1，SF 就是 1。**这比 `cmpq $0, %rax` 的编码更紧凑。**

- SET 指令的后缀命名规律与跳转指令完全相同，用于根据条件码向寄存器写入布尔值。
- 后缀由 equal、not、sign、less、greater 等单词组合而成，对照英文原词很容易记忆。

| SET 指令 | 置 1 的条件 | 数学含义 |
| :--- | :--- | :--- |
| `sete D` | ZF = 1 | 相等（Equal）或为零 |
| `setne D` | ZF = 0 | 不等（Not Equal） |
| `sets D` | SF = 1 | 负数 |
| `setns D` | SF = 0 | 非负数 |
| `setg D` | ~(SF ^ OF) & ~ZF | 有符号**大于** `>` |
| `setge D` | ~(SF ^ OF) | 有符号**大于等于** `>=` |
| `setl D` | SF ^ OF | 有符号**小于** `<` |
| `setle D` | (SF ^ OF) \| ZF | 有符号**小于等于** `<=` |
| `seta D` | ~CF & ~ZF | 无符号**大于** `>` |
| `setae D` | ~CF | 无符号**大于等于** `>=` |
| `setb D` | CF | 无符号**小于** `<` |
| `setbe D` | CF \| ZF | 无符号**小于等于** `<=` |

-  SET 的目标操作数 D 只能是**单字节寄存器**（如 `%al`、`%r8b`）或者**一个内存地址**，不能直接写入 64 位寄存器。因为 SET 本质上是在写一个布尔值（0 或 1），一个字节就足够了。如果需要把这个布尔值用于后续的 64 位运算，通常后面会跟一条 `movzbl` 把单字节零扩展到完整的寄存器。

- **有符号比较**（`setg`、`setl` 等）看 **SF 和 OF 的异或**。因为补码溢出会反转符号位的实际含义——一个减法结果本应是负数，但如果发生了溢出，SF 可能变成 0，所以需要用 OF 来修正。
- **无符号比较**（`seta`、`setb` 等）看 **CF**。无符号数没有"负数"的概念，两个无符号数比较只需要判断有没有借位（前一个是否小于后一个）。

### 3.06.3 跳转指令

- **跳转指令**直接影响程序计数器 `%rip`，让处理器从另一个地址开始执行

```asm
# 无条件跳转
jmp     Label       # 直接跳转到 Label 标号处
jmp     *Operand    # 间接跳转：从 Operand（寄存器或内存）中读出地址，跳过去
# 条件跳转
je      Label       # ZF = 1 时跳转（相等 / 为零）
jne     Label       # ZF = 0 时跳转（不等）
js      Label       # SF = 1 时跳转（负数）
jns     Label       # SF = 0 时跳转（非负）
jg      Label       # 有符号 > 时跳转
jge     Label       # 有符号 >= 时跳转
jl      Label       # 有符号 < 时跳转
jle     Label       # 有符号 <= 时跳转
ja      Label       # 无符号 > 时跳转
jae     Label       # 无符号 >= 时跳转
jb      Label       # 无符号 < 时跳转
jbe     Label       # 无符号 <= 时跳转
```

- 间接跳转 `jmp *Operand` 在 switch 语句的跳转表中使用较多。

### 3.06.4 跳转指令的编码

跳转指令在机器码中**不直接存储目标地址**，而是存储一个**偏移量**。这个偏移量是目标地址相对于**跳转指令的下一条指令地址**的差值：

```text
目标地址 = 下一条指令的地址 + 偏移量
```

这种编码方式叫 **PC 相对寻址**（PC-relative）。PC 就是程序计数器 `%rip`，它始终指向**下一条**指令。

用 objdump 看一个实际例子：

```text
  4003fa:   74 f6       je     4003f2
  4003fc:   48 89 d0     mov    %rdx, %rax 
```

机器码 `74 f6` 中，`74` 是 `je` 的操作码，`f6` 是偏移量 `f6` 作为有符号单字节 = **-10**（补码）

下一条指令地址是 `0x4003fc`，所以目标 = `0x4003fc + (-10) = 0x4003fc - 0x0a = 0x4003f2`

向前跳时偏移量为**正**，向后跳时偏移量为**负**。偏移量可以是 1、2 或 4 字节，编译器自动选最短的。

这样做的好处是代码**与位置无关**——如果整段代码被搬到内存的另一个位置，只要指令之间的相对距离不变，跳转就仍然正确，不需要链接器修正目标地址。

```asm
jmp     .L2         # 向前跳，偏移量为正
jmp     .L3         # 向后跳，偏移量为负
```

### 3.06.5 用条件控制来实现条件分支

C 中的 `if-else` 语句在汇编层面被翻译为**条件跳转**。编译器使用的基本模板非常固定，本质上就是把 C 的控制流改写为 goto 风格，然后再翻译为汇编：

```c
// C 源码
if (test-expr)
    then-statement;
else
    else-statement;
```

```asm
# goto
    if (!test-expr)
        goto else;    
    then-statement;    
    goto done;           
else:
    else-statement;      
done:
```

- 书上举了一个取差的绝对值的例子，当 x < y 时结果取 y - x，否则取 x - y：

```asm
# C: long absdiff(long x, long y)
absdiff:
    cmpq    %rsi, %rdi     
    jge     .L2         
    movq    %rsi, %rax     
    subq    %rdi, %rax     
    ret
.L2:
    movq    %rdi, %rax    
    subq    %rsi, %rax    
    ret
```

### 3.06.6 用条件传送来实现条件分支

*这一节讲了一个很重要的概念，现代处理器的**分支预测**机制，也就是说处理器在等待判断的时候自己会先猜测并执行一个分支*

- **条件传送**（CMOV）可以在一些情况同时计算两个分支的表达式，然后根据条件码**选择**其中一条结果写入目标寄存器，不需要任何跳转
- CMOV 指令的后缀命名规律与 SET/JMP 完全一致。

```asm
cmove   S, R    # ZF = 1 时传送（相等）
cmovne  S, R    # ZF = 0 时传送（不等）
cmovg   S, R    # 有符号 > 时传送
cmovge  S, R    # 有符号 >= 时传送
cmovl   S, R    # 有符号 < 时传送
cmovle  S, R    # 有符号 <= 时传送
cmova   S, R    # 无符号 > 时传送
# ... 其余 cmovX 与 jX/setX 的命名完全对应
```

- 源 S 可以是寄存器或内存，但目的 R **只能是寄存器**（同样遵循源和目的不能同时为内存的限制）。

- 用条件传送改写 `absdiff` 的逻辑是这样的：

```c
long absdiff_cmov(long x, long y) {
    long tval = x - y;     
    long rval = y - x;     
    long cmp = x >= y;      
    if (cmp) return tval; 
    else     return rval;
}
```

- 这个做法省掉了一次可能预测失败的跳转。条件传送有一个代价：两个分支的表达式**都会被求值**。如果某个分支里有函数调用或者复杂计算，全部无条件求值反而浪费，这时编译器会退回到条件跳转。
- 只有当两个分支都非常简单（通常就是做一次算术操作）时，cmov 才有优势

### 3.06.7 循环

- C 语言有三种循环语法：`do-while`、`while` 和 `for`，但在汇编层面，它们全部被翻译为**条件跳转构成的循环结构**

- **do-while** 

```c
do
    body-statement;
while (test-expr);
```
```asm
loop:
    body-statement       
    test-expr            
    jne     loop        
```

- **while** 

```c
while (test-expr)
    body-statement;
```
```asm
    goto test          
loop:
    body-statement        
test:
    test-expr            
    jne     loop        
```

- **for** 循环本质上就是 while 循环：`for (init; test; update) body` 先执行 `init`，然后等价于 `while (test) { body; update; }`，最终也会被翻译为和 while 一样的跳转结构。

### 3.06.8 switch 语句

- **switch 语句在条件密集时使用跳转表**（jump table）来做到 O(1) 的调度

假设这样一段代码：

```c
void switch_eg(long x, long n, long *dest) 
{
    switch (n) 
    {
        case 100: *dest = x + 1; break;
        case 102: *dest = x + 2; break;
        case 103: *dest = x + 3; break;
        case 104: *dest = x + 4; break;
        case 106: *dest = x + 6; break;
        default:  *dest = 0;
    }
}
```

case 的值在 100 到 106 之间，范围很小（跨 7 个数），但中间有缺失（101 和 105 没有定义）

这时编译器构建一个**跳转表**，即一个存储了 7 个地址的数组，每个地址对应一个 case 或 default 的入口：

```text
跳转表（.L4）的内容：
  索引 0 (n=100) → .L3    (case 100)
  索引 1 (n=101) → .L8    (default，因为没有 case 101)
  索引 2 (n=102) → .L5    (case 102)
  索引 3 (n=103) → .L6    (case 103)
  索引 4 (n=104) → .L7    (case 104)
  索引 5 (n=105) → .L8    (default)
  索引 6 (n=106) → .L9    (case 106)
```

生成的汇编代码先做**边界检查**（如果 n 不在 100~106 之间就直接跳 default），然后把 n 减去 100 得到表的索引，再通过一次**间接跳转**到正确的入口：

```asm
switch_eg:
    subq    $100, %rsi             
    cmpq    $6, %rsi        
    ja      .L8                
    jmp     *.L4(,%rsi,8)         
```

- `jmp *.L4(,%rsi,8)` 这条指令的含义是：以 `.L4` 为基址，以 `%rsi` 为索引（每个表项 8 字节，因为存的是 64 位地址），从内存中读出目标地址，然后跳过去 

这种实现的性能是 O(1)——不管有多少个 case，都只需要一次边界检查加一次间接跳转

但前提是 case 值**足够密集**，编译器才愿意为中间缺失的 case（如 101 和 105）也分配表项（填成 default 地址）

如果 case 值非常稀疏（比如 `case 1, case 1000, case 1000000`），跳转表会大得离谱且大部分是浪费的，这时编译器会退回到**顺序比较**或**二分查找**的方式。

*跳转表解释了 switch 在某些场景下比多层 if-else 快得多的原因。*
