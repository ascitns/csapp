## 3.09 异质的数据结构

### 3.09.1 结构

- 结构在**内存**中也是**连续**的
- 与数组相似，指向结构的**指针**就是结构**第一个**字节的地址
- 结构元素的引用也通过**偏移值**实现
- 结构有时可以看作"横着的**数组**"——但由于各字段**数据类型**不统一，可能会因**对齐**而产生空隙。

### 3.09.2 联合

- 联合用**不同**字段引用**同一块**内存。
- 所有成员都共享**同一段**内存（不是**逐个存放**），偏移量均为 **0**。
- 联合的大小至少是其**最大**字段的大小，并且会按照对齐要求向上取整。
- **联合可以用来访问不同数据类型的位模式**：联合中的不同成员**共享**同一段存储空间，因此可以通过**不同**的数据类型对同一组比特进行读取或解释。

### 3.09.3 数据对齐

- 对齐原则：任何 **K** 字节的基本对象的**地址**必须是 **K** 的倍数。
- 对于结构，为了满足对齐，有时需要在字段之间插入空隙。
- 对于结构组成的数组，还需要在首尾衔接处也考虑对齐。

| K（字节数） | 类型                |
| ----------- | ------------------- |
| 1           | char                |
| 2           | short               |
| 4           | int, float          |
| 8           | long, double, char* |

*字段顺序可以优化内存*

> SIMD 指令（SSE/AVX）为了高效处理向量数据，需要内存按 16 字节对齐，因此编译器、malloc 和栈都必须保证对齐，否则可能出错或性能下降
